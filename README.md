# Нейро-Нечеткая Система Реконструкции Нейтронных Спектров (XAI 2.0)

Проект для восстановления спектров нейтронов по показаниям детектора (сферы Боннера) с использованием адаптивной нейро-нечеткой системы (ANFIS) и объяснимого искусственного интеллекта (SHAP).

## ✅ 2026 Update (зафиксированная версия метода)

Основной рабочий метод зафиксирован как **EA-minimal** (без rank/teacher-веток в основном контуре):

```text
L = L_main + γ D(p, q_err)
```

где:
- `p` — внутренняя чувствительность/важность признаков ANFIS,
- `q_err` — error-aware важность через рост ошибки при маскировании признака:

```text
q_err,j ~ max(0, L(y, f(x^(-j))) - L(y, f(x)))
```

Ключевой смысл: важность признаков становится частью обучения, а не только post-hoc объяснением.

### Что подтверждено экспериментально

1) **SML2010 (fast, 10 seed):**
- `ΔR² mean = +7.206e-06`, `wins/losses = 10/0`
- Faithfulness:
  - Vanilla post-hoc: `top < random < bottom`
  - EA-minimal: `top > random > bottom`

2) **SML2010 (non-fast, 5 seed, random_trials=20):**

Итого: основной вклад — **функционально корректное ранжирование важности** при сохранении качества.

## 📐 Математическая формулировка

### ANFIS-модель
Пусть вход `x ∈ R^d` — отсчёты сфер Боннера `(Q1,…,Qd)`, выход `y ∈ R^m` — спектр по `m` энергетическим бинам.

База нечетких правил (для правила `r`):

```text
Rule r:
  Если x1 есть Ar1 и ... и xd есть Ard,
  то y_hat^(r)(x) = b_r + Σ_j W_{rj} * x_j
```

где `Arj` — нечеткие множества (гауссовские функции принадлежности), `b_r ∈ R^m`, `W_{rj} ∈ R^m`.

Степень срабатывания правила:

```text
w_r(x)      = Π_j μ_{Arj}(x_j)
μ_{Arj}(x)  = exp( - (x - c_{rj})^2 / (2 * σ_{rj}^2) )
```

Нормированные веса и итоговый выход:

```text
w̄_r(x)   = w_r(x) / Σ_k w_k(x)
y_hat(x) = Σ_r w̄_r(x) * y_hat^(r)(x)
```

### Этап 1: PSO (Vanilla) — целевая функция

```text
L_vanilla(Θ) = (1/N) * Σ_i || y_hat(x_i; Θ) – y_i ||_2^2 + λ * ||Θ||_2^2
```

где `Θ` — все параметры ANFIS (центры/ширины МФ и линейные коэффициенты), `λ` — коэффициент L2‑регуляризации. Глобальный поиск PSO минимизирует эту цель.

### Этап 2: SHAP-регуляризация (градиентный fine-tune)
Пусть `Φ(x) = [φ_j]_1^d` — SHAP-вектор важностей признаков для предсказания `y_hat(x)` (усреднённо по бинам). Общая функция потерь:

```text
L_total = L_main + γ * (
    α * R_sparsity
  + β * R_consistency
  + δ * R_faithfulness
  + η * R_stability
)
```

Компоненты:
- **Разреженность (sparsity)**: `R_sparsity = (1/N) * Σ_i ||Φ(x_i)||_1` поощряет малое число важных детекторов.
- **Согласованность (consistency)**: штраф за противоречивые знаки/монотонности (например, отрицательная важность при положительной ожидаемой связи с целевым бин‑выходом).
- **Достоверность (faithfulness)**: положительная связь `|Φ|` с действительным влиянием на выход (аппроксимируемым локальной чувствительностью/градиентом).
- **Стабильность (stability)**: малое изменение `Φ(x)` при малых возмущениях `x + ε`, то есть устойчивость важностей к шуму.

Эта цель оптимизируется Adam/SGD поверх PSO‑инициализации (two‑stage).

## 🧠 Почему это работае
- PSO снимает проблему инициализации в невыпуклой ANFIS (подбирает структуру правил и параметры МФ).
- SHAP-регуляризация “вживляет” интерпретируемость в модель, заставляя использовать минимально необходимый и стабильный набор детекторов, согласованный с физикой.
- Итог — комбинация точности (PSO/vanilla) и научно интерпретируемого поведения (SHAP), что важно для статей и реального применения в нейтронной спектрометрии.

## 🔬 Двухэтапный метод: подробнее и формулы

Мы решаем две фундаментальные задачи обучения ANFIS в реальной задаче реконструкции спектров:
1) надёжная инициализация параметров и структуры правил (глобальная оптимизация),  
2) внедрение интерпретируемых ограничений на использование признаков (SHAP).

Шаг 1 (Vanilla, PSO). Минимизируем классический MSE‑лосс с L2‑штрафом:

```text
L_vanilla(Θ) = (1/N) * Σ_i || y_hat(x_i; Θ) – y_i ||_2^2 + λ * ||Θ||_2^2
```

где `Θ` включает центры/ширины МФ и линейные параметры правил. PSO эффективно обходит многомодальные ландшафты и задаёт “правильную” точку старта.

Шаг 2 (SHAP fine‑tune). Добавляем к основной MSE‑целевой функции `L_main` регуляризацию по компонентам:

```text
L_total = L_main + γ * (
    α * R_sparsity
  + β * R_consistency
  + δ * R_faithfulness
  + η * R_stability
)
```

Идеи компонентов:
- **Sparsity**: `R_sparsity = E_x ||Φ(x)||_1` — поощряем малое число значимых датчиков `Q_j`.
- **Consistency**: штраф за несогласованность знака `φ_j` с физически ожидаемым влиянием (например, с эмпирическим знаком частной производной `∂ŷ/∂x_j`).
- **Faithfulness**: максимизируем согласие величины `|φ_j|` с локальной чувствительностью (градиентом) — в простейшем виде через `1 - corr`.
- **Stability**: `R_stability = E_ε ||Φ(x) – Φ(x + ε)||_2^2` при малых `ε`, устойчивость важностей к шуму.

Итог — fine‑tune сводит выбор структуры (PSO) и интерпретируемые ограничения (SHAP) в единую цель, что уменьшает склонность к “хитрым” нелокальным зависимостям и улучшает воспроизводимость.

## 🧩 Что такое SHAP‑регуляризация и как она помогает интерпретируемости
SHAP даёт аддитивное разложение вклада признаков в предсказание: `ŷ = φ0 + Σ_j φ_j`. Мы используем SHAP‑вектора `Φ(x)` не только post‑hoc, но и во время обучения как регуляризатор, который:
- делает модель разреженной (меньше датчиков — проще экспертиза и отладка);
- повышает согласованность знаков/монотоний с физикой (меньше “артефактов”);
- стабилизирует важности (меньше скачков при шуме на входе).

Практически это даёт: понятные глобальные важности (bar‑чарты), стабильные локальные объяснения и отчёты, где “видно”, за счёт каких сфер Боннера восстановлен спектр.

## 👁️ Интерпретируемость для конечного пользователя
- **Ясно, какие детекторы важны.**  
  Глобальные графики важности (особенно SHAP) подчёркивают 2–4 ключевых канала `Q_j`; остальные вносят небольшой вклад. Это даёт понятный ответ, какие сферы Боннера критичны для реконструкции спектра.
- **Понятно, как восстановлен конкретный спектр.**  
  Для выбранных образцов одновременно визуализируются:  
  (1) истинный и предсказанный спектр,  
  (2) отсчёты сфер Боннера,  
  (3) профиль важностей по детекторам. Пользователь видит, какие датчики “толкнули” модель к текущему спектру.
- **Объяснения стабильны и воспроизводимы.**  
  Для похожих спектров модель опирается на одинаковый набор детекторов; при малом шуме во входных данных SHAP‑профиль меняется плавно. Это повышает доверие к объяснениям и облегчает экспертизу.
- **Всё привязано к физике эксперимента.**  
  Объяснения даются в терминах привычных объектов: номера детекторов, их чувствительность по энергии, энергетические бины спектра. Никаких “абстрактных нейронов”: только Q‑каналы, бины и их вклады.

### Post‑hoc SHAP vs SHAP‑регуляризация (в чём отличие)
При классическом подходе “модель + post‑hoc SHAP” модель сначала обучается только на MSE, без каких‑либо ограничений на структуру зависимостей, а SHAP‑значения считаются уже после обучения. В этом случае:
- SHAP только описывает текущее поведение модели, но не мешает ей использовать странные, шумовые или нестабильные зависимости;
- две почти одинаковые выборки могут иметь сильно разные профили важности;
- интерпретируемость носит чисто диагностический характер (“посмотреть внутрь”), но не управляет тем, *как* модель учится.

В предлагаемом двухэтапном методе SHAP‑подобные важности входят прямо в функцию потерь как регуляризатор. Это означает, что:
- модели с “мусорными” важностями (слишком много детекторов, нестабильность, несогласованность с физикой) получают повышенный loss и вытесняются оптимизацией;
- модель изначально подбирается из пространства решений, где профиль важностей разрежен и устойчив;
- SHAP здесь — не только инструмент анализа, но и **ограничение** на допустимые решения. В этом смысле интерпретируемость становится не внешним слоем, а внутренним свойством обученной модели.

## 🔍 Прозрачность модели
- **Структурная прозрачность.**  
  ANFIS описывается явной базой правил “если `Q_j` в таком‑то диапазоне, то спектр такой‑то”, с параметризованными функциями принадлежности и линейными выводами. Эти правила и МФ можно просматривать и интерпретировать напрямую.
- **Функциональная прозрачность.**  
  SHAP‑регуляризация накладывает ограничения на структуру важностей признаков. Модель не просто “как‑то работает”, а обязана использовать небольшой, устойчивый и физически согласованный набор детекторов, иначе растёт штраф в функции потерь.
- **Прозрачность без внешнего “объяснителя”.**  
  В классическом подходе “модель + post‑hoc SHAP” объяснения строятся поверх уже обученного чёрного ящика. Здесь же интерпретируемое поведение встроено в этап обучения, поэтому дополнительные внешние суррогатные модели не требуются: сами правила ANFIS и SHAP‑профили уже являются основной формой объяснения.

### Прозрачность реализации (по коду)

Прозрачность здесь не только концептуальная, но и практическая: любой шаг метода можно проследить по коду и артефактам.

- **Где “живёт” модель.**

  - Второй этап (SHAP‑регуляризация) реализован в `src/models/shap_trainer_improved.py`.  
    В этом файле явно видно, как считается градиентная важность признаков, как она нормализуется и как входит в регуляризаторы sparsity / consistency / faithfulness / stability.

- **Где задаётся поведение метода.**  
  - Основной сценарий запуска двухэтапного обучения — `train.py`.  
    В нём:
    1. Загружаются реальные и, при необходимости, синтетические данные (`data_loader`).
    2. Запускается PSO‑обучение базовой ANFIS через `ANFISManager`.
    3. Запускается SHAP‑fine‑tune через `ShapAwareANFISTrainerImproved`.
    4. Считаются метрики на 100% реальных данных, выполняется денормализация и сохраняются все артефакты.
  - Конфигурация всех шагов в явном виде задана в `configs/config_integrated_shap.yaml` (число правил, параметры PSO, сила и структура SHAP‑регуляризации).

- **Как связаны код и результаты.**  
  - Каждый запуск обучения сохраняет JSON‑сводку `training_summary_<timestamp>.json` в `results/`.  
    В этой сводке зафиксированы:
    - использованный конфиг (`config_path`);
    - размеры train/shap_train/test;
    - метрики (включая по диапазонам энергий);
    - пути ко всем сохранённым файлам (предсказания, выборки, важности, история SHAP).  
  - Скрипт `plot_results.py` читает именно эту сводку и строит все графики. По нему можно проследить, как:
    - загружаются выборки `samples_*_{X,y,pred,sum}.npy`;
    - строятся спектры, ошибки, scatter‑диаграммы;
    - читаются `feature_importance_*` и `feature_importance_shap_*` и преобразуются в бар‑чарты;
    - для каждого сохранённого примера строится пара “отсчёты Боннера → спектр”.

- **Повторяемость шаг за шагом.**  
  Любой исследователь может:
  1. Запустить `train_vanilla_real_only.py` и `train.py` с текущими конфигами.
  2. Получить те же типы сводок (JSON) и файлов в `results/`.
  3. Запустить `plot_results.py --summary ...` и получить весь набор графиков.  
  Вся логика — в открытом коде, без скрытых “магических” шагов, поэтому каждый этап (от МФ до SHAP‑профилей) можно проверить, модифицировать или воспроизвести.

## 🧭 Проблематика и мотивация (почему именно так)
- Классическая ANFIS чувствительна к инициализации и может залипать в локальные минимумы → нужен глобальный поиск (PSO) на первом этапе.
- Даже хорошо инициализированная модель может использовать “лишние” датчики/нестабильные паттерны → нужен объяснимый штраф (SHAP‑регуляризация), который формирует интерпретируемые и устойчивые зависимости.
- Интегрированная (одноэтапная) схема без надёжной инициализации часто уступает в стабильности — разделение на PSO → SHAP повышает надёжность и воспроизводимость.

## 📝 Тезисы для статьи (коротко)
- Представлен двухэтапный конвейер ANFIS для реконструкции нейтронных спектров: глобальная PSO‑инициализация и SHAP‑регуляризация.
- Достигнута высокая точность (Vanilla R²≈0.864; Two‑Stage R²≈0.837) на реальных данных с интерпретируемыми зависимостями.
- SHAP использован как компонент целевой функции (не только post‑hoc), что обеспечивает разреженность по датчикам и устойчивость объяснений.
- Автоматическая визуализация спектров и графиков сфер Боннера, полные отчёты и метрики в `results/`.
